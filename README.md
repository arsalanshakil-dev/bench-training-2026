# bench-training-2026

**CodingCops · Bench Training Programme 2026**

> *One week. Every day. No excuses.*

This repository contains your pre-training work for the CodingCops Bench Training Programme. Complete everything here **before Session 1**. The work you push this week is the baseline your trainers will use to assess where you start.

|---|---|

## Repository Structure

```
bench-training-2026/
└── pre-training/
    ├── day-1/   # Python basics: variables, control flow, loops
    ├── day-2/   # Functions, lists, dicts
    ├── day-3/   # OOP + File I/O: Task Tracker CLI
    ├── day-4/   # APIs + HTTP requests
    ├── day-5/   # Pandas + data analysis (Titanic dataset)
    ├── day-6/   # Neural networks from scratch
    └── day-7/   # Mini project (your choice)
```

Each folder contains your `.py` scripts and a `README.md` with a short reflection on what you built and what you learned.

## Before You Begin: GitHub Setup

**Do this before Day 1.** Exercises don't count unless they're pushed.

1. Create a **public** GitHub repo named `bench-training-2026`
2. Create the folder structure above (push an empty `README.md` to each folder)
3. Share your repo link in **#bench-training** on Slack by Day 1 morning
4. Use descriptive commit messages: `day-1: add control flow exercises` — not `update`

**Set up SSH keys now** — stop copy-pasting passwords into the terminal.
→ [GitHub SSH setup guide](https://docs.github.com/en/authentication/connecting-to-github-with-ssh)

---

## Daily Commit Format

Each day, commit with this message format:

```
day-N: <brief description of what you built>
```

Examples:

- `day-1: data types + control flow exercises`
- `day-3: OOP task tracker with JSON persistence`
- `day-7: reddit headline analyser — mini project complete`

---

## Day 7 Mini Project Options

Pick **one** and build it properly. The mock interview will ask you to walk through it.

- **Option A** — Reddit Headline Analyser *(Reddit JSON API, word frequency, stats)*
- **Option B** — Budget Tracker CLI *(SQLite, income/expense tracking, reports)*
- **Option C** — Country Info API Wrapper *(REST Countries API, comparison CLI)*

**Non-negotiable for all options:** functions/classes, error handling, working README with example output, no dead code.

---

## Mock Interview

20 questions. A trainer picks from the list at the start of Session 1. No notes, no googling.

Topics covered: Python fundamentals, data + logic, APIs, neural network concepts, your Day 7 project.

Aim for 2 minutes per answer. If you go blank — talk through your thinking. That's better than silence.
